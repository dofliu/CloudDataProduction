# 作業範本:製程漂移偵測(半導體腔體 · subtle fault)

一份和 [預測性維護](作業範本_預測性維護.md) 互補的作業。那份是「軸承會不會壞」的**分類 / RUL** 題;
這份是「**製程悄悄漂移、良率慢慢爛**」的**迴歸 / 偵測**題 —— 沒有單一警報跳,要靠多訊號趨勢抓早期徵兆。
更接近半導體 / 精密製程現場「機台沒壞、但東西越做越差」的真實痛點。出題 / 評分都已腳本化。

## 為什麼「更難」

半導體腔體 `semi_process_chamber` 的 `process_drift` 是**指標型退化**:它**不會讓設備進 fault**,
只是把 `particle_count`(微粒 / 良率指標)一路往上推、把隱藏的製程健康度往下拉。
所以學生**等不到 fault 警報**,得自己從 `particle_count / chamber_pressure / vacuum_pump_current …`
的相關趨勢,回推**看不見的製程健康度 `gt_health_min`**(1=完全正常,0=嚴重漂移;≈良率健康)。

## 一、老師出題(3 步)

```powershell
# 1) 名冊 roster.txt(每行一個學號)
# 2) 產每學號各自的資料(train 含標籤 + 私有 test 去標籤 + 答案金鑰)
.\.venv\Scripts\python.exe tools\make_assignment.py --roster roster.txt --device chamber-01 --component process_drift
#    想同時保留一點真空泵退化當干擾(更難):加大 --degradation-scale(預設 0.12)
# 3) assignments\<學號>\ 發給學生;assignments\_answer\ 自己留(勿發)
```

- 每學號 **seed = 學號雜湊** → 漂移起始時點 / 訊號實現各不同(copy 不了、各自可重現)。
- 換 `--master` 整批換實現(重補考 / 下屆)。沿用同一支 `make_assignment.py`,不需額外工具。

## 二、學生拿到 / 要交什麼

拿到 `assignments/<學號>/`:

| 檔 | 說明 |
|---|---|
| `train_chamber-01.csv` | 訓練集,**含**隱藏製程健康度 `gt_health_min`(監督式標籤)與所有觀測訊號 |
| `test_features.csv` | 私有測試集,**只有觀測訊號**(`gt_*` 標籤被拿掉)→ 對它預測後繳交 |
| `manifest.json` | 來源溯源(seed / engine_commit) |

**任務**:訓一個迴歸模型,從觀測訊號(`particle_count`、`chamber_pressure`、`vacuum_pump_current`、
`pump_temp`、三支 `gas_flow`、`rf_power`、`throughput` …)預測**製程健康度**。
提示:`particle_count` 是最強線索,但**有雜訊且非線性**;早期 subtle 區(健康度 0.9→0.6)最難,
平滑 / 多訊號 / 特徵工程會贏過只看單一 `particle_count`。

**繳交**一份 `<學號>_pred.csv`,對 `test_features.csv` 每列預測,欄位:
```
sim_t, pred_health
```
(`pred_health` ∈ [0,1];沿用 test_features 的 `sim_t` 當對齊鍵。)

## 三、評分 rubric(總分 100)

| 項目 | 配分 | 怎麼算 |
|---|---|---|
| **離線—製程健康迴歸** | 50 | `grade_chamber_assignment.py` 的 `offline_score × 0.5`(擬合 MAE + R²,對私有測試集) |
| **線上活廠—偵測未知漂移** | 30 | 部署即時良率監測服務;老師在活廠對某腔體注入**學生沒看過、之後才發生**的 `process_drift`,量**偵測延遲**(多快發現健康度掉)與**是否誤報**(demo + 佐證截圖 / log) |
| **過程與 defend** | 20 | 特徵 / 平滑選擇理由、早期 subtle 區怎麼抓、健康門檻怎麼定;2 分鐘 defend 自己那份的數字 |

離線用自動評分:
```powershell
.\.venv\Scripts\python.exe tools\grade_chamber_assignment.py --subs-dir submissions   # submissions\<學號>_pred.csv
# → 印 健康MAE / R² / 離線分,存 assignments\grades_chamber.json
```
> 驗證過:RandomForest 基準(只用可見訊號)得 MAE 0.048 / R² 0.907 / 離線分 77 —— 可解,且留給
> 更好模型(平滑、特徵工程、抓早期區)加分空間。全猜 0.5 之類的偷懶解 R²≤0 → 低分。

## 四、線上活廠驗收(AI 幫不上的那 30 分)

離線是「建良率監測模型」,線上是「**在未知活廠即時抓到悄悄開始的漂移**」:

1. 老師起常駐世界(`.\run-engine.ps1`),公布 LAN 位址 + 6020 等埠。
2. 學生把訓好的模型包成服務,持續讀腔體遙測、**即時輸出製程健康度**(連線骨架見
   [student_kit/p0_modbus_reader.py](../student_kit/p0_modbus_reader.py),或「開始」分頁的連線包)。
3. 老師在**未告知的時點**對某腔體注入 `process_drift`(教師控制台 → 注入故障,target 選 `process_drift`);
   學生服務要在**良率明顯崩壞前**就把健康度下滑 / 微粒上升的趨勢抓出來並示警。
4. 以偵測延遲、誤報數 defend;真懂的人才在「機台沒壞、東西變差」時提早發現。

> 進階自動計分變體:同一腔體另有**會故障**的 `vacuum_pump_wear`。老師改注入真空泵退化,學生就能走
> `/api/predictions` 在泵故障前送預測、由平台自動算 lead time(同 [PdM 範本](作業範本_預測性維護.md) 第四節)。
> 一台腔體同時練「subtle 不故障漂移(迴歸)」與「會故障退化(預測)」兩種技能。
