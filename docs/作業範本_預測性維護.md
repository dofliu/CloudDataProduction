# 作業範本:預測性維護(可直接用)

一份**離線建模 + 線上活廠驗收**的 PdM 作業,每學號資料不同、私有測試集評分、設計成「只會叫 AI」拿不到分。
出題 / 評分都已腳本化。資料一致性與凍結原理見 [資料集與作業.md](資料集與作業.md)。

## 一、老師出題(3 步)

```powershell
# 1) 準備名冊 roster.txt(每行一個學號)
# 2) 產每學號各自的資料(train + 私有 test + 答案金鑰)
.\.venv\Scripts\python.exe tools\make_assignment.py --roster roster.txt --device comp-01 --component motor_bearing
#    要埋感測器陷阱(進階):加 --sensor-prob 0.3
# 3) 把 assignments\<學號>\ 整個發給對應學生;assignments\_answer\ 自己留(勿發)
```

- 每學號 **seed = 學號雜湊** → 故障時間 / 訊號實現各不同(同學 copy 不了、各自可重現)。
- 換 `--master` 整批換實現(下次開課 / 重補考用)。資料用 `course-2026S1` 引擎 tag 產、**發出後不重產**。

## 二、學生拿到 / 要交什麼

拿到 `assignments/<學號>/`:
| 檔 | 說明 |
|---|---|
| `train_comp-01.csv` | 訓練集,**含** ground-truth 標籤(`ttf_sim_s` 剩餘壽命秒、`fail_within_24h` 0/1) |
| `test_features.csv` | 私有測試集,**只有觀測訊號**(標籤被拿掉)→ 對它預測後繳交 |
| `manifest.json` | 來源溯源(seed / engine_commit) |

**繳交**一份 `<學號>_pred.csv`,對 `test_features.csv` 每列預測,欄位:
```
sim_t, pred_fail_within_24h, pred_rul_h
```
(`pred_fail_within_24h` = 0/1 或機率;`pred_rul_h` = 預測剩餘壽命,小時。沿用 test_features 的 `sim_t` 當對齊鍵。)

## 三、評分 rubric(總分 100)

| 項目 | 配分 | 怎麼算 |
|---|---|---|
| **離線—故障分類** | 25 | `25 × F1`(對私有測試集) |
| **離線—RUL 迴歸** | 25 | `25 × max(0, 1 − MAE_h/48)` |
| **線上活廠驗收** | 30 | 把模型接 `/api/predictions`,對**老師之後才注入、學生沒看過**的故障算 lead time / 命中率 / 誤報(平台自動計分,見下) |
| **過程與口頭 defend** | 15 | 特徵選擇理由、誤報 error analysis、門檻怎麼定;2 分鐘 defend 自己那份的數字 |
| **感測器陷阱(若開)** | 5 | 有沒有分辨「設備壞 vs 感測器壞」(`is_sensor_fault`),不被漂移騙去誤報 |

離線兩項用自動評分:
```powershell
.\.venv\Scripts\python.exe tools\grade_assignment.py --subs-dir submissions   # submissions\<學號>_pred.csv
# → 印 F1 / RUL MAE / 離線分,存 assignments\grades.json
```
> 驗證過:完美預測→100、偷懶(全猜不故障)→0。

## 四、線上活廠驗收(AI 幫不上的那 30 分)

離線是「建模型」,線上是「**在未知的活廠即時表現**」——這輪只有真懂的人贏:
1. 老師起常駐世界(`.\run-engine.ps1`),公布 LAN 位址 + 6020 等埠。
2. 學生把離線訓好的模型包成服務,訂閱遙測、在**故障發生前** `POST /api/predictions {device, student:<學號>, eta_sim_s, confidence}`(骨架見 [student_kit/p3_predictor.py](../student_kit/p3_predictor.py))。
3. 老師注入 / 跑災難日腳本(`disaster_day`)製造故障;平台用 ground-truth 比對,算 **lead time / 命中率 / 誤報**。
4. 即時排名在**學生面 → 競賽榜**(預測榜),或 `GET /api/predictions/scores`。

> 為什麼防 AI:每人資料不同 → copy 不了;線上故障是學生沒看過、老師之後才注入的 → 不能預先解;誤報重扣 → 亂報沒用。AI 能幫你「建」,幫不了你「在未知活廠即時表現」。
