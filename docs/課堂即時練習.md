# 課堂即時練習(教師指南)

比回家作業輕、**課堂上就做**的即時練習。適合「一般教室 + 學生只有手機」的情境:
老師一鍵佈題 → 學生用手機觀察後作答 → 平台即時批改、計入平時成績。

跟「每週情境釋出」(回家作業用)分開:週情境是設定整週的資料背景;課堂練習是**當下**
挑一台設備套一個情境、配好題目讓學生馬上答。

## 流程

1. **佈題(教師)**:教師控制台 →「📣 課堂即時練習」→ 點任一練習。平台會挑一台設備套用該情境
   (健康 / 感測器故障 / 設備退化),並記錄「佈題以來」當作進階題的資料窗。
2. **觀察 + 作答(學生)**:學生開「📣 課堂練習」分頁(手機),填座號 / 學號,看到題目後作答。
   - 登入的學生:座號欄自動帶入帳號;沒帳號的班級:直接打座號即可(匿名)。
3. **即時批改**:送出後立刻回饋分數與解說;可重答,取最佳分。
4. **看板(教師)**:面板即時顯示每題的作答人數、答對率、選項分佈。
5. **收題**:「收題 + 修復設備」把設備修回健康;或「收題(保留狀態)」留著繼續分析。

> 進階題(統計 / 相關 / 趨勢)需要一點資料,**佈題後讓平台跑一會兒**再讓學生答。

## 題目分兩層

| 層級 | 標記 | 內容 |
|---|---|---|
| simple | 👀 觀察即可 | 看狀態燈、讀即時值、直覺判斷 |
| complex | 🧮 要計算/分析 | 統計量、皮爾森相關、趨勢斜率、根因判斷、越界計數 |

複雜題重用既有的**誠實自動批改器**(對 ground-truth 容差計分,與回家作業同一套),
學生要用自己的 client / 監控台把資料讀下來算完,再把數字填回手機作答。

## 內建練習(`scenarios/classroom_exercises.yaml`)

| 練習 | 難度 | 教什麼 |
|---|---|---|
| 讀懂一台健康機 | 基礎 | 狀態、讀即時值、算平均 |
| 感測器問題還是設備壞了? | 進階 | 感測器故障 vs 設備退化的判讀(根因) |
| 設備真的在退化 | 進階 | 趨勢斜率、多訊號一致惡化 |
| 這台機的 OEE 是多少? | 基礎 | 可用率 / 表現 / 良率 / OEE |
| 兩個訊號有關係嗎? | 進階 | 皮爾森相關、正相關 vs 弱相關 |
| 有幾筆超過門檻? | 基礎 | 過濾 + 計數(異常偵測雛形) |

老師可自行在 YAML 增修:每個練習 = `setup`(對哪台設備套什麼狀況)+ `questions`
(tier / prompt / type / grade)。`grade.kind` 可為 `static`(選項/概念題)、`target`
(哪台被動手腳)、`submission`(委派 stats/oee/correlation/... 批改器)。

## API(供進階整合)

| 端點 | 權限 | 說明 |
|---|---|---|
| `GET /api/classroom/exercises` | 公開 | 練習清單 |
| `GET /api/classroom/active` | 公開 | 目前佈了哪題(學生手機輪詢) |
| `POST /api/classroom/answer` | 公開 | 作答 `{exercise, question, student, answer}` → 即時批改 |
| `POST /api/classroom/exercises/{id}/launch` | 教師 | 佈題 |
| `POST /api/classroom/stop` | 教師 | 收題(`{reset}` 決定是否修復設備) |
| `GET /api/classroom/board` | 教師 | 即時看板(答對率 / 分佈) |
| `GET /api/classroom/gradebook` | 教師 | 平時成績(每人每題取最佳分) |

## 計分

- **計入平時成績**:每位學生每題取最佳分,彙整平均;教師面「平時成績(課堂練習)」可查。
- 批改一律對 **ground-truth**(合成、帶標籤)容差計分;`root_cause` / `rul` 類不透露正解,只回饋對錯與方向。
