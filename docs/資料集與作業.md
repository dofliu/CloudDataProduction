# 資料集產生、作業出題、與凍結釋出

解決「系統更新會不會影響已發資料一致性」的工作流。核心:**離線作業用凍結的、可重現的資料集;線上活廠用即時系統。** 兩者分工見 README。

## 1. 確定性種子(可重現的地基)

`generate_dataset.py --seed N` 現在**真的可重現**:

```powershell
# 同 seed 兩次 → 位元完全相同;換 seed → 不同但各自可重現
.\.venv\Scripts\python.exe tools\generate_dataset.py --seed 42 --out dataset
```

- 主種子由 `world.master_seed`(場景 `sim.seed`,預設 20260101)推導每台設備的種子(`engine/templates/_common.py: default_seed`),**不再用 Python `hash()`**(那會每進程隨機化)。
- **每學號不同但各自可重現**:`--seed <學號雜湊>`,每個學生拿到不同實現、自己重跑得到同一份 → 同學間 copy 不了、但各自可驗證。

## 2. 出乾淨的作業題(指定故障)

要「一個月資料、裡面一個指定故障要預測」:

```powershell
.\.venv\Scripts\python.exe tools\generate_dataset.py `
  --devices comp-01 `                 # 只產這台
  --inject comp-01:motor_bearing:15 ` # 第 15 天讓馬達軸承開始劣化(gradual)
  --degradation-scale 0.12 `          # 放慢自然退化 → 只剩注入的那個故障
  --sensor-fault-prob 0 `             # 不加感測器雜訊(要考感測器陷阱再開)
  --sim-days 35 --step-min 10 --seed 1001 --out hw_S1001
```

- `--inject device:component:onset_day[:fault_type]`(可重複):在指定時間注入指定故障。
- `--devices`:只產特定設備(作業聚焦一台)。
- `--degradation-scale <1`:放慢自然退化,讓注入的故障成為唯一事件 → 乾淨單一故障題。
- 標籤現成:`ttf_sim_s`(RUL 迴歸)、`fail_within_24h`(分類)、`is_sensor_fault`(陷阱)。

> 變化題:`--sensor-fault-prob 0.3` 埋感測器漂移陷阱,考學生分辨「設備壞 vs 感測器壞」(防止盲目 fit)。

## 3. 凍結 + 版本溯源(隔離系統更新)

每份資料集的 `manifest.json` 現在記:`seed`、`engine_commit`、`engine_tag`、`injections`、`degradation_scale`。

**釋出流程**:
1. **學期初固定引擎版本**:`git tag course-2026S1`(作業都引用這個 tag 產)。
2. **產一次、凍結、發下去**:把 `<out>/` 整包發給學生(或放課程平台)。`manifest.json` 已記是哪個 commit 產的。
3. **發出後絕不重產**:CSV 檔本身就是契約;之後引擎怎麼改(修 bug、調訊號)都碰不到已發資料。
4. 真要重現:`git checkout <engine_commit>` + 同 `--seed` 同參數 → 完全相同。

**所以系統中途更新不影響已發作業** —— 凍結的 CSV 不變,新改動只進下一個 tag / 下一份作業。

## 4. 防「叫 AI 做完就結束」

固定資料 + AI 會秒解,所以分數要綁在 AI 幫不上的地方:
- **每人不同資料**(`--seed 學號`):同學互相 copy 不了,各報各的數字。
- **線上活廠驗收**:學生把離線訓好的模型接 `POST /api/predictions`,用**老師之後才注入、他沒看過的故障**算 lead time / 命中率 / 誤報(見 [student_kit/p3_predictor.py](../student_kit/p3_predictor.py))。AI 能幫你建模型,幫不了你在未知的活廠即時表現。
- **埋陷阱**:`is_sensor_fault` 感測器故障 → 盲目 fit 的會誤報;評分看有沒有分辨。
- **要過程**:特徵選擇理由、誤報 error analysis、門檻怎麼定;defend 自己那份的具體結果。

一句話:**離線 AI 可幫你「建」;線上活廠 + 每人不同資料 + 陷阱 = 只有「懂」才拿得到分。**
