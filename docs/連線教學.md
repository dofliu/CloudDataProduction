# 連線教學 — 用第三方工具連虛擬工業區

本園區的 Modbus / OPC-UA / MQTT 都是**真的協定 server**,可用任何標準客戶端連線。
本文教你用三套免費工具各驗一次。所有數據皆為**合成(synthetic)**,僅供教學。

> ⚠ 先看**設備目錄**:瀏覽器開 `http://<主機>:8077/api/catalog`,或網頁的「設備目錄」分頁。
> 每個 tag 對應哪個 register / OPC-UA node / MQTT topic,全在那裡 —— 那就是你的接線規格書。

## 1. Modbus TCP — 用 Modbus Poll(或 QModMaster)

設備的 tag 放在 **holding registers**;`float32` 佔 **2 個連續暫存器**,**big-endian**(word & byte order 都 big)。

以讀 cnc-01 的 `vibration_rms`(在 register 9)為例:

1. Connection → Connect → **Modbus TCP/IP**,IP `127.0.0.1`,Port **6020**。
2. Setup → Read/Write Definition:
   - **Slave ID = 1**(cnc-01 的 unit_id)
   - **Function = 03**(Read Holding Registers)
   - **Address = 9**,**Quantity = 2**
3. Display 格式選 **Float**,word order 選 **AB CD**(big-endian)。
   - 若數值看起來亂跳/不合理,改成 **CD AB** 再試(word order 反過來)。
4. 應看到 `vibration_rms` 在 1~2 mm/s 附近,隨時間慢慢上升。

常見 tag register(CNC):`state`=0(int16)、`spindle_current`=5、`spindle_temp`=7、
`vibration_rms`=9、`tool_wear`=11、`part_count`=17(int32)。**完整清單以目錄為準。**

> CLI 玩家:`modpoll -m tcp -p 6020 -a 1 -r 10 -c 2 -t 4:float 127.0.0.1`
> (modpoll 的 address 從 1 起算,所以 register 9 要填 10)。

### 1b. Modbus 專屬埠(multi_port,進階)

老師若開了 multi_port(`.env` 的 `MULTI_PORT_ENABLED=true`),**每台設備還會多一個自己的專屬埠**
(從 6100 起:cnc-01=6100、cnc-02=6101…,目錄 / 戰情版查得到),像真實設備一台一個 IP:port。
連專屬埠時**不用管 unit_id**(填 1 即可),register 對照與共用埠完全一樣。

- 共用埠思維:連 `:6020`,改 Slave ID 切設備(一條連線管全部)。
- 專屬埠思維:連 `:6102` 就是 comp-01,**每台一條連線**,你得自己管多條資料管線。

### 1c. 四種 object type(不是只有 holding register)

同一台設備的點位刻意分散在不同 Modbus object type,讓你練習「依規格書決定怎麼讀」。每個點位的
`object` / `fc` / `address` 都在 **/api/catalog** 與設備目錄頁列出:

| Object type | FC | 內容 | 型別 | ModScan 位址 |
|---|---|---|---|---|
| Holding Register | 03 | 量測值 | float32(2 格)/ int16 / int32 | `40001 + reg` |
| Discrete Input | 02 | 狀態旗標 `running`/`fault`/`idle`/`warning`/`heartbeat` | bool | `10001 + addr` |
| Input Register | 04 | 唯讀 int:`state_code`、量測縮放鏡像(工程單位 = 值 / `scale`) | int16 / int32 | `30001 + addr` |
| Coil | 01讀 / 05寫 | 命令 bit:`run_enable`(停機/復機)、`reset_fault`(清故障,瞬時)| bool | `00001 + addr` |

**最容易踩的雷**:holding 第 1 格(40001)是 `state`,型別是 **int16(只佔 1 格)**,
所以 float 量測是從 **第 2 格(40002)** 起成對讀。把整段當 float 從 40001 湊,會把 state 吃掉、
而且每個 float 都差半格 → 看起來「很接近卻不精準」。`state` 要用 16-bit 整數單獨讀 40001。

一支範例把三種一次讀給你看(對照儀表板):

```bash
python student_kit/p1_modbus_objecttypes.py --unit 2     # im-01 射出機
```

> `input_register` 的縮放鏡像(如 `clamping_force_x100`)是教學設計:同一個物理量,
> 這裡用 **int32 定點**放在 FC04,工程單位要自己 `÷scale`(×100 → 兩位小數);
> holding 的 float32 則是直接的工程值。兩者相除應一致。

### 1d. 命令線圈(Coil,FC01/05)—— 教師可寫,學生唯讀

每台設備有命令線圈:`run_enable`(coil 0,停機/復機)、`reset_fault`(coil 1,清故障,瞬時)。

- **學生**:用 FC01 讀線圈狀態(ModScan Point Type `01: COIL STATUS`,Address `1` = 00001 → addr 0)。
  學生埠的 FC05 寫入不生效(會被引擎下一拍蓋回)——**學生唯讀**。
- **教師**有兩條寫入路徑(「教師碼才可寫」):
  1. **REST(帶教師 token)**:`POST /api/devices/{id}/coil {"name":"run_enable","value":false}`,
     或直接在 2D 世界設備面板按「停機/復機、清除故障」。
  2. **教師控制埠**(網路隔離版):`.env` 設 `MODBUS_CONTROL_PORT=5023`,教師對該埠用 FC05 寫線圈會
     轉成引擎命令(寫 `run_enable=0` → 設備停機)。此埠只給教師(走 Tailscale / 不對學生公布)。

寫 `run_enable=0` 後,該設備會停機(state→idle、退化暫停);`reset_fault=1` 等同清故障。

兩種同時存在,可當場對比;階段一進階要求學生用專屬埠、自行管理多連線,更貼近真實場域。

## 2. OPC-UA — 用 UaExpert(Unified Automation,免費)

1. 工具列 **+**(Add Server)→ 在 Custom Discovery 輸入
   `opc.tcp://127.0.0.1:6041/clouddata/` → OK。
2. 展開該 server,**Security Settings 選 `None` / `None`**(本案匿名、無加密)→ Connect。
3. 左側 Address Space 展開 **Objects → c01 → cnc-01**,把 `vibration_rms` 拖到中間
   Data Access View,即可看到數值即時更新。
4. 想看別台設備就展開對應 folder(`c02/comp-01`、`c03/agv-01`…)。

> 連不上常見原因:選了有加密的 Security Policy。一定要選 **None**。

## 3. MQTT — 用 MQTT Explorer(免費)或 MQTTX

1. 新建連線:Host `127.0.0.1`,Port **6083**,不填帳密(匿名)→ Connect。
2. 左側 topic 樹會自動長出 `park/c01/cnc-01/state` …,點開看**整包 JSON**
   (含 `state` 與所有 `tags`),每 ~0.5 秒更新一次。
3. 想一次訂全部:topic 填 `park/#`。

> CLI 玩家:`mosquitto_sub -h 127.0.0.1 -p 6083 -t "park/#" -v`

---

## 4. 三協定讀到的值應一致

同一個 `vibration_rms` 不管用哪個協定讀,值都該差不多(只差量測雜訊與取樣時間)。
這正是核心教學點:**同一份隱藏狀態,三種協定、三種定址方式**。
網頁的「**戰情版**」分頁會一次幫你對三協定做連線自測,可拿來對照你自己工具的結果。
