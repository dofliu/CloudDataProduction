# 部署與運維(常駐 5090 主機)

把「我跑得起來」變成「無人值守跑一學期」。重點:**一定用 venv 的 Python 跑**,掛看門狗自動重啟,DB 持久化。

## 1. 一次性安裝

```powershell
cd D:\...\claudDataProduction
python -m venv .venv                                  # 若還沒建
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
copy .env.example .env                                # 填 TEACHER_TOKEN 等
```

> ⚠ 不要用裸 `python`(那是全域 Python,版本會被別的工具拉漂移)。一律 `.\.venv\Scripts\python.exe`,
> 或直接用 `run-engine.ps1`(內含版本檢查)。

前端(選用,給世界 / 目錄 / 戰情版的網頁):
```powershell
cd web; npm install; npm run build       # 正式用 build;開發用 npm run dev
```

## 2. 手動啟動 / 確認

```powershell
.\run-engine.ps1                          # 用 venv 起引擎(Modbus 6020 / OPC-UA 6041 / MQTT 6083 …)
# 另開一個視窗驗證:
.\.venv\Scripts\python.exe tools\smoke_test.py     # 11 項不變式檢查,回傳 0=全過
Invoke-RestMethod http://127.0.0.1:8077/api/health # {ok:true, devices:45, …}
```

## 3. 常駐 + 自動重啟(擇一)

### A. 看門狗 + 工作排程(內建、免額外軟體,建議)

```powershell
# 以系統管理員執行:開機自動跑 watchdog,watchdog 顧著引擎、health 失敗自動重啟
.\deploy\install-startup-task.ps1
Start-ScheduledTask -TaskName CloudDataProduction      # 立即啟動(不必等重開機)
```

`deploy\watchdog.ps1` 每 30 秒打一次 `/api/health`,連續 3 次失敗(含「卡住但沒崩」)就清埠重啟。
日誌在 `logs\watchdog.log`、`logs\engine.log`。

### B. NSSM(更像正式 Windows 服務,需下載 nssm.exe)

```powershell
nssm install CloudDataProduction powershell.exe "-NoProfile -ExecutionPolicy Bypass -File D:\...\run-engine.ps1"
nssm set CloudDataProduction AppExit Default Restart       # 崩潰自動重啟
nssm start CloudDataProduction
```

## 4. 日常運維

| 事項 | 指令 |
|------|------|
| 看健康 | `Invoke-RestMethod http://127.0.0.1:8077/api/health` |
| 跑檢查 | `.\.venv\Scripts\python.exe tools\smoke_test.py` |
| 三協定自測 | `Invoke-RestMethod http://127.0.0.1:8077/api/diagnostics/protocols` |
| 看排程狀態 | `Get-ScheduledTask -TaskName CloudDataProduction \| Get-ScheduledTaskInfo` |
| 重啟 | `Restart-ScheduledTask -TaskName CloudDataProduction` |
| 引擎日誌 | `Get-Content logs\engine.log -Tail 50 -Wait` |

## 5. 資料持久化

- 預設 `DB_BACKEND=sqlite`,歷史寫入 `historian.db`(專案根目錄,**進程重啟資料不失**)。
- 學生用 SQL 撈訓練資料:直接讀該 `.db`(任何 SQLite 工具 / `pandas.read_sql`),或打 `GET /api/history?device=&tag=`。
- 要上正式時序庫:裝好 PostgreSQL+TimescaleDB 後,`.env` 改 `DB_BACKEND=timescale`、填 `TIMESCALE_DSN`。

## 6. 對外(暫緩)

校外連線(Cloudflare Tunnel 走 HTTP、Tailscale 走原生協定)待能上 5090 主機再開;校內學生直接連 LAN IP + 6xxx 埠即可。
防火牆需放行輸入埠 6020 / 6041 / 6083(及 multiport 6100–6144)。
