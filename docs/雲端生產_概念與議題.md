# 雲端生產是什麼?——課名的兩種斷句,與課程的邊界

> 給「雲端生產數據導論」修課學生的補充講義。
> 目的:把課名講清楚、把「雲端生產(雲製造)」這個更大的題目攤開,並誠實標出**本平台教到哪、沒教到哪**。
> 讀法建議:第 0 節可當第一堂課的三分鐘思辨;第 4–6 節提供整學期可討論、可動手的題目。

---

## 0. 先想三分鐘:課名怎麼斷句?

「**雲端生產數據**」這五個字,至少能斷成兩種意思——而且兩種都對:

| 斷句 | 意思 | 重點在 | 這門課 |
|---|---|---|---|
| 雲端 ｜ 生產數據 | 把**生產數據**透過網路(雲端)擷取、傳輸、儲存、分析 | **數據** | ← 我們大部分時間在做這個 |
| 雲端生產 ｜ 數據 | 「**雲端生產**(雲製造)」這種生產模式所產生 / 需要的數據 | **生產模式** | ← 今天要補的視角 |

你目前寫的 client、算的統計、畫的趨勢、對的 OEE,都是第一種讀法:**「生產數據上雲」**。
但如果把「雲端生產」當一個詞,它其實是製造業一個明確的範式(paradigm),值得單獨認識——
因為它決定了**未來你處理的資料會長什麼樣、從哪來、要回答什麼問題**。

## 1. 我們現在在做的:生產數據上雲(數據層 / Industrial IoT)

課程主軸是**資料的一生**:

```
產生 → 用工業協定接取 → 傳輸/串流 → 儲存 → 分析/統計 → 視覺化 → KPI/決策 → 輕量預警 → 報告
```

這條線的本質是:**生產發生在本地工廠,資料被傳到雲上處理。** 機台還是那台機台、還在原地運轉;
上雲的是「它吐出的數字」。這一層在業界叫 **工業物聯網(Industrial IoT, IIoT)** 或**數據層**。

本平台就是這一層的教具:模擬引擎產生擬真、帶 ground-truth 的資料,經 Modbus / OPC-UA / MQTT 暴露,
你連上來讀、存、算。**這是導論課的正確聚焦**——先把資料的一生走通。

### 放進 ISA-95 分層看

製造業有個經典的分層模型 **ISA-95**(也叫自動化金字塔):

```
L4  ERP        企業資源規劃:接單、採購、財務、主排程      ← 「計畫層」
L3  MES        製造執行系統:工單、派工、在製追蹤、品質、稼動  ← 「執行層」
─────────────────────────────────────────────────────
L2  SCADA/監控  監視與資料擷取、HMI                        ← 我們的協定層 + 數據層在這附近
L1  感測/致動   PLC、感測器、致動器
L0  現場        實體製程(機台、產線)                      ← 模擬引擎在「假裝」這一層
```

本平台大致覆蓋 **L0–L2 的模擬 + 把資料往上送**,並且已經有 **L3(MES)的雛形**(見第 4 節)。
你寫的 client 站在 L2/L3 的位置往下讀資料——這就是「生產數據上雲」。

## 2. 另一種讀法:雲端生產(Cloud Manufacturing, 雲製造)

**雲端生產 / 雲製造(Cloud Manufacturing, CMfg)** 是一種**服務導向的網路化製造模式**。一句話:

> 把「製造資源」與「製造能力」虛擬化、封裝成雲端服務,匯集成資源池,讓需求方**按需、跨企業**地
> 取用——就像雲端運算把運算資源變成服務一樣,雲製造把**製造能力**變成服務。

這常被稱作 **製造即服務(Manufacturing as a Service, MaaS)**。跟雲端運算對照最好懂:

| 雲端運算 | 雲端生產(雲製造) |
|---|---|
| IaaS:租運算/儲存,不自己買伺服器 | 租**產能**:有訂單時調度別人的閒置機台,不自己蓋廠 |
| PaaS:平台上開發部署 | 平台上做**設計 / 模擬 / 排程 / 協同製造** |
| SaaS:用軟體不用安裝 | 用**製造服務**:上傳圖檔→媒合工廠→回件 |
| 隨用隨付、彈性擴縮 | 產能隨用隨付、跨廠彈性調度 |

### 雲製造的幾個關鍵特徵

1. **資源虛擬化與服務封裝**:一台 CNC、一條產線、一組製程知識,都被描述成「可被搜尋、調用的服務」。
2. **資源池化與共享**:很多廠的閒置產能匯成一個池,跨企業共用。
3. **按需自助、隨用隨付**:需求方下單即媒合,用多少付多少。
4. **分散式與協同**:一個產品可能由分散各地的多家工廠**協同**完成。
5. **覆蓋全生命週期**:設計→模擬→生產→測試→管理→維護,都能是雲上服務。
6. **平台是核心**:靠一個**工業互聯網平台**把供需、資源、交易撮合起來。

### 真實世界長這樣

雲製造不是紙上概念,它落地成 **工業互聯網平台(Industrial Internet Platform)**:
海爾 COSMOPlat、樹根互聯 RootCloud、西門子 MindSphere、GE Predix、PTC ThingWorx 等。
它們對外提供設備接入、資料分析、產能協同、甚至產能交易的服務。

### 「數據上雲」 vs 「生產上雲」

| | 數據上雲(IIoT / 我們在做的) | 生產上雲(雲製造) |
|---|---|---|
| 上雲的是 | 設備吐出的**資料** | **製造能力**本身 |
| 生產在哪 | 還在本地工廠 | 由雲平台**跨廠調度** |
| 平台賣什麼 | 資料擷取 / 分析 / 監控 | **產能 / 製造服務**(MaaS) |
| 主要問題 | 這台機況如何?趨勢?KPI? | 這張單交給誰做最划算?產能怎麼撮合? |
| 是不是本課主線 | ✅ 是 | ⬜ 視野延伸 |

**重點**:雲製造**建立在**數據上雲之上——沒有可信的設備資料,產能就無法被描述、調度、結算。
所以你現在練的資料功夫,正是雲製造的地基。

## 3. 誠實的邊界:本平台涵蓋什麼、不涵蓋什麼

呼應專案鐵則「**數據必須誠實**、絕不誇大」,這裡把邊界講白:

| 雲製造要素 | 本平台 | 說明 |
|---|---|---|
| 設備接入(多協定) | ✅ 有 | Modbus / OPC-UA / MQTT,channel-mux 多設備 |
| 帶標籤的可信資料 | ✅ 有 | 合成但誠實、帶 ground-truth |
| 資料儲存 / 歷史查詢 | ✅ 有 | Historian(SQLite / TimescaleDB) |
| 分析 / KPI / OEE | ✅ 有 | `/api/oee`、繳交自動批改 |
| **MES(工單 / 稼動 / 排程雛形)** | 🟡 雛形 | `engine/mes.py`;見第 4 節 |
| 多租戶 / 跨廠資源池 | ⬜ 無 | 目前是「一人一廠」的單一園區,不是產能池 |
| 產能目錄 / 訂單媒合 / 產能交易 | ⬜ 無 | 沒有「把單發給別廠競標」這種市集 |
| 跨廠協同製造 / 結算 | ⬜ 無 | 一單綁一台,不做跨廠路由 |

**結論**:本平台是「**雲端生產數據**(數據上雲)」的完整教具,**不是**一個完整的「雲製造平台」。
這不是缺點——導論課本就該先把資料層練扎實。把缺的那幾格當成**視野**與**期末發想題**(第 7 節)即可。

## 4. 雲端生產管理(Cloud Production Management ≈ Cloud MES)

你問「是否要加入類似 **MES** 的雲端生產管理」——好消息是:**平台已經有 MES 雛形了**,
只是還沒被當成一個獨立的學習主題攤開。

### MES 是什麼

**MES(Manufacturing Execution System,製造執行系統)** 是 ISA-95 的 **L3 執行層**,
夾在 ERP(計畫、接單)與現場控制(PLC/SCADA)之間。它回答的是:

> 「這張訂單現在做到哪?哪台機在做?會不會準交?稼動率多少?品質如何?出問題能不能追溯?」

典型職能:**工單管理、派工排程、在製品(WIP)追蹤、品質、稼動 / OEE、追溯**。

**雲端 MES(Cloud MES)** = 把 MES SaaS 化、多廠共用——這一步就把「生產管理」接回了「雲製造」。

### 平台現況:已有的 MES 能力

`engine/mes.py` 已實作 **Phase 1 MES**:

- **工單驅動運轉**:每間公司有訂單佇列;設備「有單才開工」,不再一開機就永遠轉。
- **稼動率(order_density)**:教師可設 light/normal/high → 待機空檔多寡改變產出、退化、OEE。
- **完工換單、維持 backlog**:世界常保有活的設備與**連續可分析的歷史**。
- **對外唯讀視圖**:`GET /api/orders`(工單)、`GET /api/oee`(排程基準 OEE)。

這代表你不只能分析「一台機的訊號」,還能分析「**一間廠的生產**」——這正是生產管理的資料題材。

### 目前刻意留白的(可當進階題材)

`WorkOrder` 已帶交期 `due_t`,但 **Phase 1 僅顯示、尚未評分**。也就是說:

- **準交率(On-Time Delivery)**、**WIP 水位**、**瓶頸辨識**、**排程情境對 OEE 的影響**
  這些「生產管理」指標,平台有**原始資料**(工單的 released/started/done/due + 完工數),
  但還沒幫你算好——**留給學生自己從 `/api/orders` 算**,正好是好練習(見第 6 節)。

## 5. 給課程的討論議題(可課堂辯論 / 短報告)

1. **課名思辨**:我們到底在學「雲端｜生產數據」還是「雲端生產｜數據」?兩種讀法對資料的需求差在哪?
2. **數據上雲 vs 生產上雲**:畫出 ISA-95 金字塔,標出本平台覆蓋哪幾層、缺哪層。IIoT 與雲製造的界線在哪?
3. **產能為什麼能當服務賣?** 要把「一台機的產能」變成雲上可交易的服務,資料上至少要先能誠實回答哪些問題?
4. **雲製造的資料信任問題**:跨企業共享產能時,大家憑什麼相信對方回報的機況 / 良率 / 交期?
   (呼應本課「資料誠信、帶 ground-truth」的鐵則。)
5. **雲端 MES 的好處與風險**:MES 上雲、多廠共用有何效益?又有什麼資安 / 資料主權 / 延遲的顧慮?
6. **把本平台想像成雲製造平台**:還缺哪些模組(多租戶、產能目錄、訂單媒合、跨廠調度、結算)?
   哪些是「資料層就能支撐」、哪些要新的服務層?

## 6. 給課程的動手練習(多數用現有平台就能做)

> 標「可自動批改」者可走 `POST /api/submissions`;標「自算」者資料平台都有,只是要自己算。

| 練習 | 做什麼 | 用到 | 對應主題 |
|---|---|---|---|
| **生產節奏觀察** | 抓一台 producer 設備的 `state`,算「運轉 : 待機 : 故障」時間占比 | `/api/history`、`/api/devices` | 稼動 / MES |
| **工單分析** | 從 `/api/orders` 撈自己公司工單,算平均生產工時、完工數、目前 WIP(未完成單數) | `/api/orders` | 生產管理 |
| **準交率自算** | 用工單的 `due_sim_t` vs `done_sim_t` 算 On-Time Delivery %(平台沒幫你算,自己算) | `/api/orders` | 生產管理(進階) |
| **稼動率情境對照** | 教師切 order_density(light/normal/high),你比較同一台機在三情境下的 OEE / 產出 | `/api/oee`、`/api/course/status` | KPI / 實驗設計 |
| **瓶頸辨識** | 多台設備比較產出與稼動,指出全廠瓶頸並用資料佐證 | `/api/orders`、`/api/oee` | 生產管理(進階) |
| **OEE 自算對帳** | 純用你接取的資料自算 OEE(可用率×表現×良率),跟 `/api/oee` 對帳 | `/api/history`、`/api/oee` | 生產分析(已在 W11) |
| **產能發想(書面)** | 假設要把你公司的產能掛上雲製造平台,設計「產能服務卡」需要哪些欄位 / 資料 | 設計題 | 雲製造(視野) |

## 7. 建議如何融入 18 週課程

不必大改課綱,建議以「視角補充」方式嵌入(對應 [課程規劃_18週.md](課程規劃_18週.md)):

- **W1 課程導論**:加入第 0 節「課名斷句思辨」當開場,順帶用一張圖講清楚**數據上雲 vs 生產上雲**。
- **W11 生產數據與 KPI**:本來就講 OEE / MES 工單;在此補「MES 是 ISA-95 L3」「雲端 MES → 雲製造」的
  視角,並開放第 6 節的**工單分析 / 準交率 / 瓶頸**當進階練習。
- **期末專題(W15→W18)**:新增一個**可選發想題**——「把我的單廠資料平台,往雲製造平台再走一步」:
  學生不必真的實作跨廠交易,而是**用資料論證**「要支撐產能上雲,資料層還缺什麼、我能補上哪一塊」。

### 若要讓「生產管理」更好練,平台可選配套(待確認再做)

目前準交率 / WIP / 瓶頸都能「自算」,但若要像其他作業一樣**自動批改**,可再加:

1. **`GET /api/orders` 擴充生產管理 KPI**:回傳 on-time %、平均 WIP、平均前置時間(平台算好 ground-truth)。
2. **新增作業型別 `production`**:學生 POST 準交率 / WIP,平台對 ground-truth 容差自動計分
   (比照現有 `oee` / `events`,見 [課程規劃_18週.md](課程規劃_18週.md) 的作業型別表)。

> 這兩項屬平台增修,不在本講義範圍;需要的話再開一個 PR 實作。

## 名詞速查

- **雲端生產 / 雲製造(Cloud Manufacturing, CMfg)**:製造資源與能力虛擬化為雲服務、按需跨企業調度的製造模式。
- **製造即服務(MaaS)**:把製造能力當服務賣,對照雲端運算的 IaaS/PaaS/SaaS。
- **工業物聯網(IIoT)**:設備聯網、資料上雲的那一層;本課主線。
- **工業互聯網平台**:雲製造的載體(COSMOPlat / RootCloud / MindSphere / Predix…)。
- **MES(製造執行系統)**:ISA-95 L3 執行層,管工單 / 排程 / 在製 / 品質 / 稼動 / 追溯。
- **ERP**:ISA-95 L4 計畫層,接單 / 採購 / 財務 / 主排程。
- **ISA-95 / 自動化金字塔**:L0 現場 → L1 感測致動 → L2 監控(SCADA)→ L3 MES → L4 ERP。
- **OEE**:設備綜合效率 = 可用率 × 表現 × 良率。
- **WIP(在製品)**:已開工未完工的工單 / 數量。
- **準交率(On-Time Delivery)**:如期完工的工單占比。

> 本講義為課程視野補充,平台資料為**合成(synthetic)**、帶 ground-truth,明確標示為合成數據,絕不對外宣稱是真實場域量測。
